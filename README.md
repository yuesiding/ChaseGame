**基于 DDA 优化的跨平台光线追踪迷宫游戏**。

======================== 游戏演示 ===========================
============================================================

**游戏逻辑**
玩家出生在地图一侧，目标是找到地图另一侧的蓝色地板区域。每一关的地图尺寸都会增大。
**操作方式**
WASD 控制移动，鼠标控制视角（可选 QERF 辅助操作）
ESC 键退出游戏
每一帧根据键盘、鼠标输入更新玩家位置和视角，仅当目标位置无墙体时，玩家才能移动。

**关卡搭建** 
即生成指定尺寸的地图：
先生成随机位置、随机属性的方块，地图边缘固定为全高棱柱墙体
随机游走算法尝试打通至地图另一侧，过程中移除方块形成通路
到达对岸后，将该位置标记为关卡出口
每个新关卡随机生成纹理

===================== 核心技术讲解 ==========================
============================================================

**亮点1：跨平台兼容架构设计**
##基于olcPixelGameEngine单文件跨平台引擎开发，一套代码可在 Windows、Linux 三大系统直接编译运行
##适配了不同平台的编译链路：Windows 端 MinGW 编译、Linux 端 X11 图形支持，无平台锁死
##相比传统单平台课程项目，具备更强的工程移植性与通用性，可直接在多平台部署运行

**亮点2：基于DDA算法的光线追踪渲染优化**
// ==============================
在最初的基础版本中，我们采用**传统小步长迭代检测光线**，逻辑简单但效率极低，大量运算浪费在空白网格的无效检测上。为此，我们重点引入了**DDA数字微分分析算法**进行核心优化。
// ==============================

**核心原理**
基础实现方式是反向追踪光线：从相机出发发射光线，与游戏场景产生交互，核心包含三类基础光线：
   视线路径：从相机发射的初始光线，返回光线碰撞物体的坐标
   反射光线：视线路径撞击反射面后生成的次级光线，新方向为入射光线相对于表面法向量的反射方向，支持多次反弹（反射中嵌套反射）
   阴影光线：从视线路径的碰撞点出发，朝向光源方向发射的次级光线

本游戏所有3D画面，全部依靠**自研软件光线追踪**实现。我们为屏幕每一个像素发射独立光线，构建了完整的光线交互体系，包含视线路径、反射光线、阴影光线三类核心光线。

同时我们实现了多材质物理效果，可以识别普通墙体、反射墙体、球体几何体，支持镜面多次反射、纹理贴图、实时软阴影等效果，3D视觉效果完整且流畅。

**参考资料**https://lodev.org/cgtutor/raycasting.html

除此之外，我们实现了**随机游走地图生成算法**，每一关、每一局游戏都会随机生成全新的迷宫地形、墙体纹理和地图结构，保证了游戏的随机性和可玩性，关卡递进逻辑完整，项目完成度极高。
